# Format Domain (core:format)

Last verified: 2026-05-27

## Purpose
Persists a notebook library to a single SQLite file (.forestnote) using SQLDelight. One library file holds `folder → notebook → page → (stroke, text_box)` (folders are an optional hierarchy; notebooks/folders at the root have a NULL parent). Provides auto-save on pen-up, full restore on app launch, folder/notebook/page CRUD + active-context switching, Library card/thumbnail read paths, soft-delete (Recycle Bin) of notebooks/folders, and the global `Settings` blob + per-page template overrides — all behind `NotebookRepository`.

## Contracts
- **Exposes**: `NotebookRepository` (open/forTesting/openExisting; stroke ops: saveStroke→Unit, loadStrokes, deleteStroke(String), applyErase, clearPage; text-box ops: saveTextBox(TextBox) (create or in-place move/resize/edit, upsert by ULID), loadTextBoxes()/loadTextBoxesForPage(id)→List<TextBox>, deleteTextBox(id) (SOFT-delete); notebook/page: listNotebooks, listPagesForCurrentNotebook, countPages(id)→Long, currentNotebookId, currentPageId, switchPage, switchNotebook, createNotebook(name, folderId?=null)→id, renameNotebook, deleteNotebook (SOFT-delete: standalone tombstone), bulkDeleteNotebooks(ids) (each a standalone tombstone, NULL batch), modifiedAtOf(id), createPage→id, deletePage→Boolean; folders: createFolder(name, parentFolderId?)→id, renameFolder, deleteFolder(id) (SOFT-delete cascade: tapped folder + descendants + contained LIVE notebooks share one deleted_batch_id, deleted_root_id = tapped folder), getFoldersForParent(parentId?), listAllFolders, findFolder(id)→FolderMeta?, folderPath(id), descendantFolderIds(rootId); recycle bin (E3/E4): recycleBinEntries()→List<BinEntry>, recycleBinCount()→Int, restoreEntry(BinEntry) (notebook→original folder if live else root; folder→whole batch atomically), permanentlyDelete(BinEntry), emptyRecycleBin(), purgeExpiredBinEntries(now) (called from bootstrap; honors Settings.recycleBinRetentionDays); Library reads: listNotebookCardsInFolder(folderId?), listFolderCardsForParent(parentId?), firstPageIdForNotebook(id)→id?, countStrokesForPage(id)→Long, loadStrokesForPage(id); settings: settings()→Settings, updateSettings((Settings)→Settings)→Settings, setPageTemplate(pageId, PageTemplate?, Int?); close), public types `NotebookMeta(id, name, createdAt, modifiedAt)` / `PageMeta(id, createdAt, template, templatePitchMm)` / `FolderMeta(id, name, sortOrder, createdAt, modifiedAt, parentFolderId?)` / `NotebookCard(id, name, createdAt, modifiedAt, pageCount)` / `FolderCard(id, name, parentFolderId?, notebookCount)` / `Settings` (@Serializable JSON blob) / `PageTemplate` enum, `StrokeSerializer` (encode/decode), `FolderPathLogic` (pure object: `path`/`descendants` over in-memory FolderMeta, cycle- and depth-guarded), `RecycleBinLogic` (pure object: `entries`/`batchMemberIds`/`restoreFolderFor` over in-memory `DeletedFolder`/`DeletedNotebook` rows — batch-top selection, inside-counts, restore-landing reconciliation; `BinKind`/`BinEntry` types)
- **Guarantees**: On open, bootstraps ≥1 notebook with ≥1 page and restores the active notebook+page from `app_state` (falling back to the first if the recorded ids are stale). Never zero LIVE notebooks: deleting the last live one bootstraps a fresh one (live = non-tombstoned; live queries read the `notebook_live` view). Notebook + folder deletes are SOFT (stamp `deleted_at`/`deleted_batch_id`/`deleted_root_id`; rows + their pages/strokes stay in the DB for restore); folder delete cascades the whole subtree into one atomic batch. Page + stroke deletes are now SOFT too (sync, Phase 1): `deletePage` tombstones the page AND its strokes, `clearPage`/eraser/`applyErase`-removed set `stroke.deleted_at`; live reads filter `deleted_at IS NULL`. Recycle-bin **permanent** delete + empty/purge still HARD-delete (local reclaim). A notebook always keeps ≥1 LIVE page (deleting the only live page is refused). Corrupted databases are deleted and recreated. Stroke round-trip is lossless.
- **Expects**: Android Context for database creation. `Stroke`/`StrokePoint` types from `core:ink`. The UI goes through `app:notes` `NotebookStore`, never this class directly.

## Dependencies
- **Uses**: `core:ink` (Stroke, StrokePoint types), SQLDelight (AndroidSqliteDriver), kotlinx.serialization (Settings JSON; serialization Kotlin plugin applied at module level via `settings.gradle` pluginManagement — `build-logic` conventions untouched)
- **Used by**: `app:notes` (NotebookRepository)
- **Boundary**: Must not depend on `app:notes` or Android UI

## Key Decisions
- SQLDelight over Room: type-safe SQL, smaller footprint, no annotation processing
- BLOB encoding for points: 5 ints per point (x, y, pressure, tsHigh, tsLow), little-endian. Compact and fast.
- Single library file `default.forestnote` holding `folder → notebook → page → stroke`; an `app_state` singleton row (id=0) records the active notebook+page, plus `settings_json` (the `Settings` blob) and `clipboard_json` (persisted clipboard; column added in v5, wiring deferred)
- Settings stored as ONE JSON column (`app_state.settings_json`), not a key/value table: read on launch, written rarely, never queried by field. `Settings.json` codec uses `ignoreUnknownKeys` + `encodeDefaults` so the blob round-trips across build versions and adding a field never needs a migration
- Per-page template lives on `page.template`/`page.template_pitch_mm`. **Freeze-at-creation (B4):** pages always store a CONCRETE template — the global default seeds only at creation (new notebook's first page from `Settings`; later pages copy their predecessor's columns). Changing the global default never retroactively changes existing pages. Legacy `template IS NULL` pages are baked to the current default once on open (`bakeNullPageTemplates`, idempotent). NULL is a transient/legacy state only; the resolver (`page.template ?: default`) remains as a safety fallback
- `PRAGMA foreign_keys` is OFF, so `ON DELETE CASCADE` is inert — multi-row HARD deletes (page delete; future E4 permanent-delete) run through `db.transaction { }` children-first
- Soft-delete (Recycle Bin, AC7): notebook/folder deletes stamp `deleted_at`/`deleted_batch_id`/`deleted_root_id` rather than removing rows. Every query that must ignore tombstones reads the `notebook_live`/`folder_live` VIEWs (not the base table) so the filter can't be forgotten. Standalone notebook delete → NULL batch/root; folder delete → fresh ULID batch shared across the subtree, `deleted_root_id` = the tapped folder. Subtree is computed from LIVE folders BEFORE stamping (they're still live then). The folder cascade only re-tombstones LIVE notebooks (`softDeleteNotebooksInFolders … AND deleted_at IS NULL`) so an already-standalone-deleted notebook is NOT absorbed into the folder's batch
- Recycle Bin reads/ops live on the same `db.transaction` discipline: `recycleBinEntries` feeds tombstoned rows to `RecycleBinLogic`; `restoreEntry` clears tombstone columns (folder batch: folders first so notebooks reconcile against now-live folders via `RecycleBinLogic.restoreFolderFor`); `permanentlyDelete`/`emptyRecycleBin`/`purgeExpiredBinEntries` hard-delete (strokes→pages→notebook, then folders). `Settings.recycleBinRetentionDays` (0 = never) drives the launch-time purge from `bootstrap()`
- Factory methods (open/forTesting/openExisting) instead of public constructor: controls lifecycle

## Invariants
- After open there is always ≥1 notebook, each with ≥1 page, and a populated `app_state`
- StrokeSerializer rejects truncated BLOBs (returns empty list, never crashes)
- Notebook/page/stroke ids are client-minted ULIDs (TEXT); strokes carry their id from creation, so saveStroke() does not return or change it
- Strokes load ordered by the explicit `z` column (ascending = paint order); saveStroke assigns z = max(z)+1 per page
- Stroke ops are scoped to `currentPageId`; switching notebook/page updates the current ids and persists them to `app_state`

## Key Files
- `NotebookRepository.kt` - Storage facade (notebook/page/stroke CRUD, switch, bootstrap, app_state restore)
- `StrokeSerializer.kt` - Binary point encoding/decoding
- `Settings.kt` - `@Serializable Settings` data class (global default template/pitch + startView + sync/AI/CalDAV URLs + `recycleBinRetentionDays` (0 = never) + text-box style `textFontName`/`textFontSizeV`, all defaulted) + `PageTemplate`/`StartView` enums + the shared `Settings.json` codec
- `RecycleBinLogic.kt` - Pure bin grouping/reconciliation over in-memory `DeletedFolder`/`DeletedNotebook`: `entries` (batch tops, newest-first, inside-counts), `batchMemberIds`, `restoreFolderFor` (original folder if live else root); `BinKind`/`BinEntry` types
- `FolderPathLogic.kt` - Pure folder-hierarchy logic over in-memory `FolderMeta` rows: `path` (root-first ancestor chain) + `descendants` (BFS); both cycle- and MAX_DEPTH-guarded so a corrupt hierarchy can't hang the caller
- `notebook.sq` - SQLDelight v10 schema (folder/notebook/page/stroke/text_box/app_state; `text_box` table = z-ordered text element on a page (virtual-coord x/y/width/height + text/font_name/font_size/color/weight/border_width + `z` band + `deleted_at` tombstone; `getTextBoxesForPage`/`upsertTextBox`/`softDeleteTextBox` + sync `syncRowTextBox`/`allTextBoxIds`/`applyUpsertTextBox`); `folder` table + `notebook.folder_id`; `notebook.modified_at`; `app_state.settings_json`/`clipboard_json`; `page.template`/`template_pitch_mm`; soft-delete `deleted_at` on folder/notebook AND page/stroke — folder+notebook also have `deleted_batch_id`/`deleted_root_id`; sync-only tables `sync_state`/`outbox`/`sync_row_meta`; `notebook_live`/`folder_live` VIEWs) and queries (soft-delete mutations + `softDeletePage`/`softDeleteStroke`/`softDeleteStrokesForPage`/`upsertStroke`; recycle-bin reads/writes; folder/notebook/page/stroke CRUD with live reads filtering `deleted_at IS NULL`; Library card reads; sync capture queries `getSyncState`/`ensureSyncState`/`setSyncSiteId`/`bumpNextOpSeq`/`insertOutbox`/`upsertRowMeta`/`syncRow{Notebook,Folder,Page,Stroke}`/`all{Folder,Notebook,Page,Stroke}Ids`)
- `SyncWire.kt`/`SyncOp.kt`/`SyncMerge.kt` - Sync (UltraBridge) pure core. `SyncWire` encodes a row's synced columns to wire `cols` JSON (color→unsigned int64, points→base64) AND decodes them back (`decode{Notebook,Folder,Page,Stroke,TextBox}` → typed `*Row` holders; the inverse — unsigned int64→signed DB Long, base64→BLOB — round-trips per `SyncWireDecodeTest`); `SyncOp`/`TablePK` are the wire op model; `SyncMerge` is the deterministic row-level LWW merge (`less` on `(wall_ts,op_seq,site_id)`, `normalize`, `merge`, `knownCols`) — MUST match UB's `reconcile.go` and pass `src/test/resources/sync-vectors/` (17 vectors mirrored verbatim from the UB repo — edit there, mirror here)
- Sync apply + send side (Phase 4) live on `NotebookRepository`: `applySyncOps(ops)` merges RELAYED ops by the same LWW rule (key vs `sync_row_meta`, winners upserted via `applyUpsert*` bypassing the outbox, provenance re-stamped, owning notebook `modified_at` bumped to `max(current, wall_ts)`); `pendingOps()`/`markAckedThrough(through)`/`syncCursor()`/`setSyncCursor(c)`/`syncSiteId()`/`syncAckedOpSeq()` are the send-loop accessors the `core:sync` engine drives. The coroutines engine/transport itself lives in `core:sync`, not here.
- `migrations/` - `1.sqm`/`2.sqm` DESTRUCTIVE resets; `3.sqm` adds `notebook.modified_at`; `4.sqm` settings/clipboard + per-page template; `5.sqm` `folder` table + `notebook.folder_id`; `6.sqm` soft-delete cols/indexes + re-creates the live views; `7.sqm` (v7→v8) adds `page`/`stroke` `deleted_at` + the sync-only tables; `8.sqm` (v8→v9) adds `sync_state.joined` (the initial-join-complete flag); `9.sqm` (v9→v10) adds the `text_box` table. `3.sqm`–`9.sqm` are non-destructive (no view touch)

## Gotchas
- NotebookRepository.open() silently deletes and recreates on any database error
- Ids are client-minted ULIDs minted at construction — there is no "unsaved" id state
- Schema is at version 10 (auto-derived from the nine `.sqm` files); `migrations/2.sqm` is a DESTRUCTIVE v2→v3 reset (drops existing data), consistent with `1.sqm`; `3.sqm`–`9.sqm` are non-destructive
- Text-box sync is GATED OFF: `NotebookRepository.TEXT_BOX_SYNC_ENABLED = false` so `text_box` mutations don't enqueue outbox ops yet (the UltraBridge server doesn't know `text_box` → would 409 on the schema-hash gate). The wire/merge/apply plumbing (`SyncWire.textBoxCols`/`decodeTextBox`, `SyncMerge.knownCols["text_box"]`, `buildCols`/`writeWinningOp`/`backfillOutbox` cases) is all present but dormant. Turning sync on = flip the flag + bump `SyncProtocol.SCHEMA_HASH` to match the server (coordinated)
- Sync capture (UltraBridge, Phase 2): when sync is ENABLED (a `site_id` is minted in `sync_state` via `enableSync()`), every mutating repo method appends a full-row-UPSERT op to `outbox` and stamps `sync_row_meta` provenance INSIDE the mutation's own transaction (`enqueueOp`). Sync OFF (no site_id) ⇒ every capture is a no-op, so sync ships dormant. `enableSync()` backfills one op per existing row. NOT captured: `modified_at` touches (not a synced column), permanent-delete/purge paths (local reclaim), and bootstrap/recovery inserts (deferred to the Phase-4 initial-sync handshake to avoid a fresh device uploading an auto-created notebook before it pulls)
- VIEWs (`notebook_live`/`folder_live`) are NOT auto-added to migrations by SQLDelight — only to `Schema.create()`. Any migration that introduces or must preserve a view has to `CREATE VIEW` it explicitly, or upgrading devices hit "no such table: …_live" at runtime (regression-tested in `MigrationTest.v6ToV7…`)
- `folder`/`notebook` row lookups use SQLite `IS ?` (not `= ?`) when scoping by parent/folder id so a NULL bind matches root-level rows; `= ?` would never match NULL
- `upsertAppState` is an `ON CONFLICT(id) DO UPDATE` upsert, NOT `INSERT OR REPLACE` — REPLACE would delete+reinsert the row and reset `settings_json`/`clipboard_json` to their defaults on every page switch (regression-tested in SettingsStorageTest)
- `notebook.modified_at` is bumped (via `touchCurrentNotebook`) inside every ink-mutating transaction — saveStroke, deleteStroke, applyErase (so app-layer cut/delete/paste/move inherit it), clearPage — using an injectable clock
- Do not rely on FK cascade for deletes — `PRAGMA foreign_keys` is off; use transactional children-first deletes
