# Notes App (app:notes)

Last verified: 2026-05-27

## Purpose
Main application module that wires together ink rendering, storage, and user interaction into a single-activity note-taking experience optimized for e-ink. Includes a nav bar for moving between pages, plus a full-screen Library overlay for browsing folders and notebooks as a card grid.

## Contracts
- **Exposes**: `MainActivity` (entry point), `DrawView` (drawing surface), `ToolBar` (tool selection), `PageNavigationLogic` (pure page index/bounds/label math), `ToolSelectionLogic` (pure tool/variant state), `LassoSelectionLogic` (pure selection geometry), `Clipboard`/`InProcessClipboard`, `SelectionMenuView`, `SettingsView` (full-screen Settings overlay), `SettingsFormLogic` (pure pitch-radio rules), `LibraryView` (full-screen Library overlay; `Callbacks.onOpenSearch` + public `navigateToFolder(folderId?)`), `RecycleBinView` (full-screen Recycle Bin overlay), `LibraryAdapter`/`LibraryItem` (sealed `Folder`/`Notebook`), `BreadcrumbView`, `LaunchLogic`/`BreadcrumbLogic`/`NotebookNameParser`/`RelativeTime`/`ThumbnailCacheLogic` (pure helpers), `ThumbnailRenderer`/`ThumbnailCache`/`ThumbnailLoader`, `TextBoxEditor` (in-place EditText overlay), `TextBoxMenuView` (Edit/Options/Delete pill), `FontCatalog` (/system/fonts enumeration + fallback), `search.SearchDialog` (modal AlertDialog over the Library) + `search.SearchResultsAdapter`, `ocr.OcrTextDialog` (editor modal showing server-recognized text with a forward-compatible "Device recognized" stub entry in the source dropdown)
- **Guarantees**: Strokes auto-save on pen-up. Strokes restore on app launch (non-blocking async load), reopening the last-active notebook+page; if there is no resumable notebook (blank/empty active id or empty library, per `LaunchLogic`) the app opens into the Library instead. Backend lifecycle managed across pause/resume. Finger input rejected (stylus/eraser only). Page/notebook switching commits the current stroke (via clearAll) and refreshes the canvas to avoid e-ink ghosting. Lasso selects strokes by centroid; Cut/Copy/Delete via a floating pill + in-process clipboard; Paste is tap-to-place (arm, then tap the canvas); a closed selection can be dragged to move it in place. The Library lists folders-then-notebooks inside the current folder (null = root) as a 4-column card grid; tap a notebook to open it, tap a folder to descend, breadcrumb/back to walk up.
- **Expects**: `core:ink` for backend and stroke types, `core:format` for persistence.

## Dependencies
- **Uses**: `core:ink` (InkBackend, BackendDetector, PageTransform, Stroke, Tool, StrokeGeometry, PressureCurve), `core:format` (NotebookRepository + NotebookMeta/PageMeta/FolderMeta/NotebookCard/FolderCard/FolderPathLogic, owned exclusively by `NotebookStore`), `core:sync` (SyncEngine/SyncConfig/SyncStatus + HttpUrlTransport, driven by `SyncController`), `androidx.recyclerview` (Library card grid only), kotlinx-coroutines (sync controller scope)
- **Used by**: Nothing (top-level app module)
- **Boundary**: Should not contain domain logic; delegates to core modules

## Key Decisions
- Activity (not Fragment/Compose): minimal overhead for single-screen e-ink app. Secondary screens (Settings, Library) are full-screen overlay Views added to `android.R.id.content`, not new Activities — keeps the single `NotebookStore` (single-writer DB invariant) and avoids fragments. The Library is reached by tapping the fixed-width Library grid icon in the nav bar (a fixed icon, NOT the variable-width notebook name, so different name lengths never reflow the bar; the old `showNotebookPicker` dialog is REMOVED); its own folder/notebook/create dialogs are driven by `MainActivity` via `LibraryView.Callbacks`
- **Toolbar caption convention (both the editor toolbar and the Library header):** every action is a vertical icon-over-caption cell where the cell is the clickable hitbox and the inner `ImageButton` is non-clickable (taps fall through). A trailing **`▾`** on the caption means "opens a chooser/dialog" (editor: `Fountain ▾`/`Stroke ▾`/`Template ▾`; Library: `Folder ▾`/`Notebook ▾`); a plain caption is a direct action (`Lasso`/`Clear`/`Refresh`/`Settings`/`Up`). Captions are static strings (fixed widths → no reflow). The editor toolbar is 30dp; the Library header is 44dp (icon+caption + breadcrumb needs the extra room)
- RecyclerView is used only by the Library card grid (`androidx.recyclerview` — the app's first and only RecyclerView). Thumbnails render off-thread (`ThumbnailLoader`) into an LRU `ThumbnailCache`; eviction/sizing math is pure in `ThumbnailCacheLogic`
- DrawView owns offscreen bitmap: all strokes rendered to bitmap, blitted to screen in onDraw. A single `composeStaticBitmap()` defines page layering — template → BOTTOM-band text boxes → ink → TOP-band text boxes — and every rebuild path funnels through it (onSizeChanged/mergeLoadedStrokes/mergeLoadedTextBoxes/clearAll/redrawBitmap); `setTemplate(effectiveTemplate, pitchMm)` repaints on change
- **Text boxes (Tool.Text):** z-ordered text elements rendered with `StaticLayout`, clipped to the box (overflow retained in the model, not drawn). Drag-to-draw places a box; an in-place transparent `EditText` (`TextBoxEditor`, hosted in the `canvas_container` FrameLayout that wraps DrawView) edits it WYSIWYG — wraps to width, auto-grows downward. `editingBoxId`/`transformingBoxId` boxes are skipped in the static bitmap (shown live by the EditText / an onDraw overlay). The editor commits on canvas-touch-while-editing, tool/page/notebook switch, library open, and onPause (MainActivity drives `textBoxEditor.commit()`). Tap a box to select (corner handles + `TextBoxMenuView` pill); drag interior = move, drag handle = resize (reflows live). Empty commit discards. Font/size come from the Text-cell chooser (font list + size chips, mirroring the pen-width popup), backed by `FontCatalog` (off-thread `/system/fonts` scan; resolves a stored basename to a Typeface with a default fallback so a synced box whose font is absent still renders). Active style persists in `Settings.textFontName`/`textFontSizeV`. Text-box SYNC is gated off client-side (see core:format) — boxes persist + render locally now Template pitch uses true panel PPI (293 on the Mini, set in MainActivity by isEInk), NOT densityDpi/xdpi
- Hardware eraser (TOOL_TYPE_ERASER) always erases regardless of toolbar selection
- 900ms postDelayed invalidate after pen-up: triggers quality e-ink refresh without excessive redraws
- All persistence flows through `NotebookStore` (single background thread); DrawView/MainActivity never touch `NotebookRepository` or the DB directly. Save is fire-and-forget; erase reconcile + load run off-thread and post results back to the main thread

## Invariants
- All coordinates in DrawView: virtual for storage, screen for bitmap rendering
- WritingBufferQueue released in onPause, reacquired in onResume (shared device resource)
- clearAll() only clears in-memory state; database clearing is caller's responsibility (via `store.clear`)
- Tool selection is pure logic in ToolSelectionLogic (testable without Android)
- `DrawView.mergeStrokes` is a pure companion function: loaded ink first, then session ink, deduped by ULID — so the async load never clobbers strokes drawn during the load gap
- `MainActivity.onDestroy` calls `store.shutdown()`, which drains pending saves before closing the driver
- Page/notebook index math is pure in `PageNavigationLogic` (testable without Android); `MainActivity` drives all switching/CRUD through `NotebookStore` and never touches `NotebookRepository` directly
- `goToPage`/`goToNotebook`/`reloadCurrentPage` share one idiom: `clearAll()` → `store.switch*`/`load` → `mergeLoadedStrokes` + `fullRefresh` + refresh indicator/label

## Key Files
- `MainActivity.kt` - Lifecycle management, backend + NotebookStore wiring, nav-bar wiring (page picker; notebook label opens the Library), Library callback wiring + folder/notebook dialogs, launch-into-Library decision (`LaunchLogic`), crash handler
- `NotebookStore.kt` - Single background-thread owner of NotebookRepository; async load/save/erase/clear, text-box load/save/delete (`loadTextBoxes`/`saveTextBox`/`deleteTextBox`), notebook/page list-switch-CRUD (`switchNotebook`, `switchPage`, and `switchNotebookToPage(notebookId, pageId, onLoaded)` for Library-search page-level routing), folder CRUD + Library card reads (`listNotebookCardsInFolder`/`listFolderCardsForParent`/`folderPath`) + thumbnail source wrappers, library `search(query, onResult)` aggregator (off-thread wrapper over `NotebookRepository.search`), settings load/update + per-page setPageTemplate, drain-on-shutdown. Sync (Phase 5): an `onDb` executor→coroutine bridge exposes a `SyncLocalStore` for the engine + suspend join bridges (`syncMintSiteId`/`syncBackfillOutbox`/`syncIsPristineBootstrap`/`syncDiscardBootstrapNotebook`/`syncNotebookIds`/`syncCurrentNotebookId`/`syncSettings`) — all still on the single DB writer thread
- `Logging.kt` - On-device file logging for the SSH/Termux debug loop (the locked device gives Termux no logcat access to the app). `LogFormatter` (pure, UTC grep-friendly lines) + `FileLogger` (appends gated lines to `/sdcard/ForestNote/forestnote.log`, rotates one generation at a size cap, never throws; dir injected for tests). Gated by `Settings.debugLogging` (default off, toggled in Settings "Debug" section). MainActivity builds it (primary `/sdcard/ForestNote`, fallback app-private `filesDir/logs`) and feeds its `log` lambda into `SyncController` → `SyncEngine`/`HttpUrlTransport`, which emit request/response/error detail
- `SyncController.kt` - The UltraBridge sync trigger layer (Imperative Shell): builds a `SyncEngine`+`HttpUrlTransport` from persisted Settings each session (so credential/URL edits take effect), serializes sessions via a `Mutex`, retries `Retryable` with `SyncBackoff`, runs the pull-first join handshake (`enableAndJoin`: mint → pull → discard untouched bootstrap if the server delivered content → backfill → push), drives the periodic timer, and exposes `StateFlow<SyncStatus>`. `resume()`/`pause()` are the lifecycle hooks (MainActivity onResume/onPause + Settings close); `syncNow()` is the manual button. Sync stays dormant when no server URL is configured (`SyncConfig.from` → null). `resume()` keeps running the full (idempotent) `enableAndJoin` until the persisted `sync_state.joined` flag is set — so a first attempt that failed mid-handshake (no network, wrong password) still uploads pre-sync content on a later resume instead of degrading to a plain `runSession`
- `DrawView.kt` - Touch handling, bitmap rendering, stroke/erase logic, and text-box place/edit/select/move/resize (Tool.Text); `composeStaticBitmap()` is the single layering point; `drawTextBox(canvas, box)` serves both the static bitmap and the live transform overlay
- `TextBoxEditor.kt` - In-place editing: a transparent multi-line EditText over the box in `canvas_container`; reports final text + pixel height back to `DrawView.commitTextBox`
- `TextBoxMenuView.kt` - Floating Edit/Options/Delete pill for a selected box (mirrors SelectionMenuView; non-focusable so drag-to-move/resize touches reach the canvas)
- `FontCatalog.kt` - Off-thread `/system/fonts` enumeration; `resolve(name, weight)→Typeface` with system-default fallback for empty/absent fonts
- `ToolBar.kt` - UI toolbar with tool button state management (incl. Paste enable/armed state, Template action cell → per-page template picker, OCR action cell → opens the recognized-text viewer; greyed via `setOcrEnabled` when the active page has no live `page_text_from_server` row). The Fountain cell opens a **pen-settings popup** (A10): variant rows + a 5-chip width strip (chips drawn as thickness samples); tapping a variant/width updates the popup in place (no dismiss), tap-outside dismisses. Exposes `setOnPenWidthSelected`/`loadPenWidths`/`activePenWidthLevel`/`currentPenWidthLevels`/`setOnOcrClicked`/`setOcrEnabled`. The legacy Refresh cell was removed — the device's Quick Settings exposes a full-screen GC clear that does the same thing, and the toolbar's internal `fullRefresh()`/`gcRefresh()` calls fire automatically on page/notebook switch anyway
- `ToolSelectionLogic.kt` - Pure tool selection state machine (pen/erase variants + Lasso + **per-variant pen width level**, default M: `selectPenWidth`/`activePenWidth`/`penWidthFor`/`setPenWidthForVariant`/`allPenWidthLevels`)
- `PenWidthSettings.kt` - Pure bridge: `Settings.penWidthLevels` (`Map<String,String>`) ↔ `Map<PenVariant,PenWidthLevel>` (decode drops unknown names; consumer defaults to M)
- `LassoSelectionLogic.kt` - Pure selection geometry (ray-cast pointInPolygon, integer centroid, selectedIds, bounds, translate)
- `Clipboard.kt` - `Clipboard` interface + `InProcessClipboard` (listener-based; B1 re-backs it with `app_state.clipboard_json`)
- `SelectionMenuView.kt` - Floating action pill (PopupWindow) for a lasso selection; non-focusable so drag-to-move touches reach the canvas
- `SelectionActionLogic.kt` - Pure dialog-text rules for the pill's Recognize (F1) / To-do (F2) placeholder actions: message varies by whether the endpoint URL (`selectionRecognitionUrl` / `caldavServerUrl`) is configured. No network call yet — `MainActivity.showSelectionAction` loads Settings off-thread and renders the `Dialog(title,message)`
- `SettingsView.kt` - Full-screen Settings overlay (B2): inflated over the editor via `addContentView` onto `android.R.id.content`, reusing MainActivity's single `NotebookStore` (a 2nd Activity would open a 2nd DB connection and break single-writer serialization). Editor template+pitch radios persist on change; Sync/AI/CalDAV URL fields commit on blur or IME Done. Reached via a Settings action in the Library; system back dismisses it (`MainActivity.onBackPressed`)
- `SettingsFormLogic.kt` - Pure pitch-radio rules (presets 5/7/10mm, visible iff template != BLANK, nearest-preset snap, index clamp)
- `TemplateGeometry.kt` - Pure template geometry (interior `lineOffsets(extent,pitch)`) + effective-config resolution (`page override ?: global default`, AC8.4)
- `res/layout/view_settings.xml` + `res/values/styles.xml` - Settings layout (header Back/title + scrolling sections) and its field styles (black-on-white, e-ink friendly)
- `PageNavigationLogic.kt` - Pure page navigation (index, prev/next bounds, "N / M" label, can-delete)
- `res/layout/toolbar.xml` - The eight tool cells: Fountain / Lasso / Text / Erase / Paste / Clear / OCR / Template (each whole cell is the hitbox). `Text ▾` opens the font/size chooser; OCR opens the recognized-text viewer (greyed when the active page has no server-OCR row yet)
- `ocr/OcrTextDialog.kt` + `res/layout/dialog_ocr_text.xml` + `res/drawable/ic_ocr.xml` - Editor's recognized-text viewer: a modal AlertDialog with a Spinner picking the source (Server / Device — device is a forward-compatible stub showing "not available yet"), a meta line (model · ocr_at), and a scrollable selectable TextView. Re-loads on every open via `NotebookStore.loadPageTextFromServer(pageId)`. `MainActivity.refreshOcrButtonState()` re-queries on page/notebook switch and on every `SyncStatus.Synced` so the toolbar gate reflects newly-arrived OCR rows without a manual refresh
- `res/layout/activity_main.xml` - navbar + a `canvas_container` FrameLayout wrapping DrawView (so the text-box EditText can overlay the canvas)
- `res/layout/dialog_page_template.xml` - Per-page template picker (B4): "Use default" (snapshots current default) + Blank/Dot/Ruled/Grid + pitch sub-radio
- `res/layout/navbar.xml` - Unified top bar: notebook label / prev (◀) / "N / M" indicator / next (▶) / `<include>` of `toolbar.xml`; a 1dp divider in `activity_main.xml` separates it from the canvas. `toolbar.xml` (root id `@id/toolbar`) is included here rather than placed at the bottom.
- `res/layout/dialog_notebook_properties.xml` - Long-press → Notebook Properties (name + Created/Modified/Pages)
- `RecycleBinView.kt` + `res/layout/view_recycle_bin.xml` - Full-screen Recycle Bin overlay (E3): mirrors `SettingsView` (`show/hide/isShowing`, reuses the single `NotebookStore`), opened from the Library header's now-enabled Recycle cell (which shows a "Recycle (N)" count badge), system-back closes it (`MainActivity.onBackPressed`, before the Settings/Library checks). Rows are built programmatically into a ScrollView (not a RecyclerView) — each bin top (standalone notebook or folder batch) with Restore / Delete-forever; header has Empty Bin. Folder soft-delete is reachable via the Folder Properties **Delete** button (`MainActivity.confirmDeleteFolder` → `store.deleteFolder`). E4 retention: a "Auto-empty after N days" field in Settings (`Settings.recycleBinRetentionDays`), purged at launch in the repo's `bootstrap()`
- `LibraryView.kt` - Full-screen Library overlay (C3a/C4/C5): a 4-column RecyclerView grid of folders-then-notebooks in the current folder (`currentFolderId`, null = root). Like `SettingsView`, an overlay View (not an Activity) reusing the single `NotebookStore`. Tap a notebook → open; tap a folder → descend; breadcrumb/back → walk up; Search cell in the header (icon-only, no `▾` since it's a direct action) opens the search dialog. Data via `NotebookStore`; folder/notebook dialogs + Settings + Search via `LibraryView.Callbacks` driven by `MainActivity`. Public `navigateToFolder(folderId?)` lets the search dialog jump the (already-showing) Library into a folder hit
- `search/SearchDialog.kt` + `search/SearchResultsAdapter.kt` + `res/layout/dialog_search.xml` + `res/layout/row_search_hit.xml` + `res/layout/row_search_truncated.xml` + `res/drawable/ic_search.xml` - Library search dialog: a modal `AlertDialog` with an `EditText` (IME `actionSearch`) + a `RecyclerView` of result rows (icon + title + bolded-match snippet via `SpannableString`/`StyleSpan(BOLD)`, ~56dp tall to fit a phone-sized dialog on the Mini). Adapter renders all four `SearchHit` kinds through one row layout plus an italic "truncated" footer when any branch capped at `NotebookRepository.SEARCH_DEFAULT_LIMIT`. Submit (magnifier button or keyboard IME) calls `NotebookStore.search`; stale post-dismiss callbacks are dropped via a `dialog` snapshot check. Tapping a result dismisses first, then routes via `SearchDialog.Callbacks` → `MainActivity` (`navigateToFolder` / `goToNotebook` / `goToNotebookPage`). `onPause` dismisses to avoid a window leak
- `LibraryAdapter.kt` + `LibraryItem.kt` - RecyclerView adapter over `sealed LibraryItem (Folder(FolderCard) | Notebook(NotebookCard))`; binds thumbnails via `ThumbnailLoader`/`ThumbnailCache`
- `LaunchLogic.kt` - Pure: `shouldOpenLibraryOnLaunch(activeNotebookId?, notebookCount)` — open the Library at launch only when there is no notebook to resume into (C6)
- `BreadcrumbLogic.kt` + `BreadcrumbView.kt` - Pure breadcrumb trail math + its top-bar View for the current folder path (C5)
- `NotebookNameParser.kt` - Pure parse of a notebook display name
- `RelativeTime.kt` - Pure "modified N ago" formatting for cards
- `ThumbnailRenderer.kt` / `ThumbnailCache.kt` / `ThumbnailCacheLogic.kt` / `ThumbnailLoader.kt` - Off-thread notebook-thumbnail render (first page's strokes) + LRU cache; `ThumbnailCacheLogic` holds the pure sizing/eviction math
- `res/layout/view_library.xml` + `res/layout/item_folder_card.xml` + `res/layout/item_notebook_card.xml` - Library grid + its folder/notebook card cells

## Gotchas
- ViwoodsBackend cast in onResume: only safe because isEInk guards it
- Crash handler writes to /sdcard/Download/ first, falls back to app filesDir
- E-ink mode disables animations and ripple effects to prevent ghosting
